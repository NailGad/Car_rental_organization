# 1. ПОСТАНОВКА ЗАДАЧИ

**Цель работы:** Разработать десктопное многооконное приложение на языке C++ в среде Qt Creator, реализующее функционал предметной области по варианту.

**Требования:** Приложение должно содержать минимум 5 классов, отражающих сущности предметной области, поддерживать работу с несколькими пользователями. Обязательно использование минимум двух паттернов проектирования из следующих: Observer, Strategy, Factory, Command, Composite. Данные приложения (пользователи, сущности, настройки) должны сохраняться локально (JSON, XML, или SQLite).

**Проект должен содержать:**

1. Учет автомобилей: марка, модель, статус.
2. Заказ аренды.
3. Оформление аренды: клиент, даты, стоимость.
4. Расчет по дням, штрафы за просрочку.
5. Поиск по марке, дате, клиенту.
6. Отчеты по доходу.
7. Импорт/экспорт данных.

**Рекомендуется использовать следующие сущности:**

- User — клиент/администратор (разный интерфейс)
- Car — автомобиль (марка, модель, статус)
- Rental — аренда (даты, стоимость)
- Fine — штраф за просрочку
- ReportManager — доход, загруженность

**Рекомендуемые паттерны:**

- Strategy — расчёт стоимости (день/неделя/штраф)
- Observer — обновление статуса авто при возврате

## 2.2 SWT-АНАЛИЗ ВЫБРАННЫХ ЗАДАЧ

| Задача | Действие | История | Проблема | Пути решения |
| --- | --- | --- | --- | --- |
| Оформление аренды автомобиля клиентом | Выбор автомобиля, указание дат начала и окончания аренды, подтверждение заказа | Ожидается подтверждение оформления аренды и расчет стоимости | Выбор недоступного автомобиля или пересечение дат с существующими арендами | Проверять доступность автомобиля в указанный период перед оформлением аренды |
| Добавление нового автомобиля администратором в систему | Создание записи с маркой, моделью, статусом и ценой за день | Ожидается подтверждение добавления автомобиля | Дублирование автомобиля с теми же характеристиками | Проверять уникальность комбинации марки и модели, выводить предупреждение при дублировании |
| Поиск аренд по диапазону дат администратором | Указание начальной и конечной даты для поиска аренд | Ожидается отображение списка аренд, попадающих в указанный диапазон | Неправильный порядок дат (начальная дата больше конечной) | Автоматически менять местами даты, если начальная больше конечной |

## 2.3 GOMS-АНАЛИЗ ВЫБРАННЫХ ЗАДАЧ

**Оформление аренды автомобиля клиентом**

- Goals: оформить аренду автомобиля
- Operators: движение мыши, нажатие кнопок, ввод дат
- Methods: выбор автомобиля из списка доступных, указание дат начала и окончания аренды, подтверждение заказа, проверка доступности автомобиля
- Selection rules: последовательность действий выбирается системой в зависимости от доступности автомобиля и корректности введенных дат

**Добавление нового автомобиля администратором в систему**

- Goals: добавить новый автомобиль в систему
- Operators: движение мыши, нажатие кнопок, ввод текста
- Methods: ввод марки автомобиля, ввод модели автомобиля, выбор статуса из списка, указание цены за день, сохранение записи
- Selection rules: если не все поля заполнены или цена некорректна, показывать предупреждение

**Поиск аренд по диапазону дат администратором**

- Goals: найти аренды в указанном диапазоне дат
- Operators: движение мыши, нажатие кнопок, выбор дат из календаря
- Methods: выбор начальной даты, выбор конечной даты, нажатие кнопки поиска, просмотр результатов
- Selection rules: если даты не выбраны, показывать все аренды; если выбрана только одна дата, использовать её как начальную или конечную в зависимости от контекста

## 2.4 ИССЛЕДОВАНИЕ ПРОИЗОВДИТЕЛЬНОСТИ ЭКРАННЫХ ФОРМ ПО МЕТОДУ ДЖ. РАСКИНА

- Форма «Оформление аренды» оптимизирована по принципам минимизации ошибок и визуальной последовательности полей (выбор автомобиля → даты → подтверждение).
- Форма «Управление автомобилями» оформлена в виде таблицы для понятного и привычного для пользователя управления.
- Форма «Поиск аренд» использует интуитивно понятные поля выбора дат с календарным всплывающим меню.

Константы времени операций:

- K = 0.2 с — нажатие клавиши
- P = 1.1 с — перемещение курсора
- H = 0.4 с — перемещение руки между устройствами
- M = 1.35 с — ментальная подготовка

### ЗАДАЧА 1: Оформление аренды автомобиля клиентом

**Цель:** Выбрать автомобиль из доступных, указать даты аренды, оформить заказ

**Подцели:** 
1. Зайти в систему под своим пользователем
2. Выбрать автомобиль из списка доступных
3. Указать даты начала и окончания аренды
4. Подтвердить оформление аренды

1. Авторизация в системе:
   - H (перемещение руки к клавиатуре)
   - K×10 (ввод логина, ~10 символов)
   - P (перемещение курсора к полю пароля)
   - K×8 (ввод пароля, ~8 символов)
   - P (перемещение курсора к кнопке "Войти")
   - K (нажатие кнопки "Войти")
   - M (ожидание загрузки интерфейса)
   **Итого: H + K×18 + P×2 + M = 0.4 + 0.2×18 + 1.1×2 + 1.35 = 6.65 с**

2. Выбор автомобиля:
   - P (перемещение курсора к таблице автомобилей)
   - M (выбор подходящего автомобиля)
   - K (клик по строке автомобиля)
   - P (перемещение курсора к кнопке "Забронировать")
   - K (нажатие кнопки)
   **Итого: P×2 + M + K×2 = 1.1×2 + 1.35 + 0.2×2 = 3.95 с**

3. Оформление аренды:
   - M (понимание формы аренды)
   - P (перемещение курсора к полю начальной даты)
   - K×3 (выбор даты начала, ~3 клика)
   - P (перемещение курсора к полю конечной даты)
   - K×3 (выбор даты окончания, ~3 клика)
   - M (проверка корректности дат)
   - P (перемещение курсора к кнопке "Подтвердить")
   - K (нажатие кнопки)
   - M (ожидание подтверждения)
   **Итого: M×3 + P×3 + K×7 = 1.35×3 + 1.1×3 + 0.2×7 = 9.35 с**

**Общее время выполнения задачи: 6.65 + 3.95 + 9.35 = 19.95 с ≈ 20 с**

### ЗАДАЧА 2: Добавление нового автомобиля администратором

**Цель:** Добавить новый автомобиль в систему с указанием всех характеристик

1. Открытие формы добавления:
   - P (перемещение курсора к кнопке "Добавить автомобиль")
   - K (нажатие кнопки)
   - M (ожидание открытия формы)
   **Итого: P + K + M = 1.1 + 0.2 + 1.35 = 2.65 с**

2. Заполнение формы:
   - M (понимание структуры формы)
   - P (перемещение курсора к полю "Марка")
   - K×6 (ввод марки, ~6 символов)
   - P (перемещение курсора к полю "Модель")
   - K×5 (ввод модели, ~5 символов)
   - P (перемещение курсора к списку статусов)
   - K (выбор статуса)
   - P (перемещение курсора к полю "Цена")
   - K×4 (ввод цены, ~4 символа)
   - M (проверка корректности данных)
   **Итого: M×2 + P×4 + K×16 = 1.35×2 + 1.1×4 + 0.2×16 = 9.5 с**

3. Сохранение:
   - P (перемещение курсора к кнопке "Добавить")
   - K (нажатие кнопки)
   - M (ожидание подтверждения)
   **Итого: P + K + M = 1.1 + 0.2 + 1.35 = 2.65 с**

**Общее время выполнения задачи: 2.65 + 9.5 + 2.65 = 14.8 с ≈ 15 с**

### ЗАДАЧА 3: Поиск аренд по диапазону дат администратором

**Цель:** Найти все аренды, попадающие в указанный диапазон дат

1. Выбор начальной даты:
   - P (перемещение курсора к полю "С даты")
   - K (открытие календаря)
   - M (выбор даты)
   - K×2 (навигация по календарю, ~2 клика)
   - K (подтверждение выбора)
   **Итого: P + M + K×4 = 1.1 + 1.35 + 0.2×4 = 3.25 с**

2. Выбор конечной даты:
   - P (перемещение курсора к полю "По дату")
   - K (открытие календаря)
   - M (выбор даты)
   - K×2 (навигация по календарю, ~2 клика)
   - K (подтверждение выбора)
   **Итого: P + M + K×4 = 1.1 + 1.35 + 0.2×4 = 3.25 с**

3. Выполнение поиска:
   - P (перемещение курсора к кнопке "Найти")
   - K (нажатие кнопки)
   - M (ожидание результатов)
   **Итого: P + K + M = 1.1 + 0.2 + 1.35 = 2.65 с**

**Общее время выполнения задачи: 3.25 + 3.25 + 2.65 = 9.15 с ≈ 9 с**

**Вывод:** Интерфейс оптимизирован для быстрого выполнения основных операций. Время выполнения задач находится в приемлемых пределах (9-20 секунд).

### ИССЛЕДОВАНИЕ ФОРМ ПО МЕТОДУ ДЖ. РАСКИНА

**ФОРМА 1: Оформление аренды**

Варианты выбора: список доступных автомобилей (предположим 10).

I = (1/10) × log₂(10) ≈ 0.332 бит

Вывод: выбор автомобиля для аренды создаёт умеренную когнитивную нагрузку, пользователю необходимо различать несколько автомобилей по марке, модели и цене.

**ФОРМА 2: Регистрация пользователя**

Варианты выбора: поле 1 (ввод логина пользователя), поле 2 (ввод пароля пользователя), поле 3 (выбор роли пользователя (клиент, администратор)), поле 4 (подтверждение пароля).

I = (1/4) × log₂(4) = 0.25 бит

Вывод: процесс регистрации нового пользователя очень прост, форма содержит минимально необходимое количество полей.

**ФОРМА 3: Добавление нового автомобиля**

Варианты выбора: поле 1 (ввод марки автомобиля), поле 2 (ввод модели автомобиля), поле 3 (выбор статуса автомобиля (Доступен, В аренде, На обслуживании, Зарезервирован)), поле 4 (ввод цены за день).

I = (1/4) × log₂(4) = 0.25 бит

Вывод: процесс добавления нового автомобиля очень прост, форма интуитивно понятна и содержит все необходимые поля для полного описания автомобиля.

# 5. ВЫВОД

В ходе выполнения курсовой работы были выполнены все поставленные задачи и разработана программа для работы агентства по сдаче автомобилей в аренду.

**Основные достижения:**

1. **Реализовано многооконное приложение** с раздельными интерфейсами для клиентов и администраторов
2. **Создано более 15 классов**, отражающих сущности предметной области и обеспечивающих функциональность приложения
3. **Реализованы два паттерна проектирования:**
   - **Strategy** — для гибкого расчета стоимости аренды (по дням, неделям, месяцам) и штрафов
   - **Observer** — для автоматического обновления статуса автомобилей при изменении аренд
4. **Использована база данных SQLite** для локального хранения всех данных
5. **Реализован полный функционал:**
   - Учет автомобилей с различными статусами
   - Оформление и управление арендами
   - Автоматический расчет стоимости и штрафов за просрочку
   - Гибкий поиск по различным критериям
   - Формирование отчетов по доходу и статистике
   - Импорт и экспорт данных в формате JSON

**Архитектурные решения:**

- Применена многослойная архитектура с разделением на модели, сервисы, паттерны и UI
- Реализованы сервисы для инкапсуляции бизнес-логики (CarService, UserService, RentalService, RentalSearchService)
- Использован паттерн Singleton для DatabaseManager
- Обеспечена независимость от системной даты через использование региональной временной зоны

**Извлечен опыт:**

- Проектирования интерфейсов с учетом принципов человеко-машинного взаимодействия
- Разработки программ с графическим интерфейсом с использованием Qt Framework
- Применения паттернов проектирования для создания гибкой и расширяемой архитектуры
- Работы с базами данных SQLite через Qt SQL модуль
- Разделения логики и представления для обеспечения поддерживаемости кода

Приложение готово к использованию и может быть расширено дополнительным функционалом в будущем.
